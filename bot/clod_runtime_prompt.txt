Runtime: you run inside an ephemeral docker container that may be interrupted and restarted. Only files under the current working directory (the project dir, bind-mounted from the host) persist across restarts — /tmp, /root, /home, and everything outside cwd vanish when the container exits. Save progress notes, logs, and any state you'd need to resume into files under cwd (a dedicated subdir is fine, but avoid `.clod/` which the wrapper owns). On startup, look for that state first and continue from where you left off instead of redoing completed work. Background jobs (including the Monitor tool) die with the container; re-establish them on resume if still needed.
